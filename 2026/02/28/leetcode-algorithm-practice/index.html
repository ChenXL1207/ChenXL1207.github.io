第二阶段：二分查找

二分查找

主要分为三种情况：闭区间、左闭右开、开区间，从中介绍三种写法

  • 闭区间，二分查找第一种写法

    区间的定义这就决定了二分法的代码应该如何写，因为定义target在[left, right]区间，所以有如下两点：（一定要注意【left，right】这个区间，很重要>）

    • while (left <= right) 要使用 <= ，因为left == right是有意义的，所以使用 <=

    • if (nums[middle] > target) right 要赋值为 middle - 1，因为当前这个nums[middle]一定不是target，那么接下来要查找的左区间结束下标位置就是 middle - 1

    其中提到了left==right是有意义的，为什么有意义呢? 看完灵神的B站视频后解释如下：

    由于需要更新Left与Right的区间，对于闭区间，其实Left应当更新为middle+1 或 right应当更新为middle-1，因为无论是对于Left往左，即【0，left】区间，还是对于right往右，即对应【right，nums.size（）-1】区间，都是已经搜索过的，所以其实最后L和R指针会相遇，而相遇的这个位置是有意义的，因此需要设置成while (left <= right) ，代码示例如下：

  • 左闭右开，二分查找第二种写法，如果说定义 target 是在一个在左闭右开的区间里，也就是[left, right) ，那么二分法的边界处理方式则截然不同需要注意两点

    • while (left < right)，这里使用 < ,因为left == right在区间[left, right)是没有意义的
    • if (nums[middle] > target) right 更新为 middle，因为当前nums[middle]不等于target，去左区间继续寻找，而寻找区间是左闭右开区间，所以right更新为middle，即：下一个查询区间不会去比较nums[middle]

匿名函数知识补充（Lambda 表达式）

  1. Lambda 表达式语法
部分 代码 含义
捕获列表 [……] 比如[&] 按引用捕获外部所有变量（比如 nums、threshold 都能在函数体内用）
参数列表 (int m) 函数的输入参数，和普通函数一样（这里接收一个整数 m）
返回值类型 -> bool 函数返回值是布尔类型（可省略，编译器会自动推导，但显式写更清晰）
函数体 {…} 函数的逻辑（计算 sum，判断是否超过 threshold，返回 bool）
变量 auto check = 把匿名函数赋值给变量 check，后续可像普通函数一样调用：check(5)

重点学习捕获列表：

捕获列表是 [] 里的内容，决定函数体内能使用哪些外部变量，常见写法：

写法 含义
[&] 按引用捕获所有外部变量（修改函数体内的变量会影响外部，无拷贝开销）
[=] 按值捕获所有外部变量（函数体内是拷贝，修改不影响外部）
[&nums, threshold] 按引用捕获 nums，按值捕获 threshold（精准控制，推荐）
[] 不捕获任何外部变量（函数体只能用参数和局部变量）

枚举右，维护左

对于 双变量问题，例如两数之和 $a_i$​+$a_j$​=$t$，可以枚举右边的 $a_j$​，转换成 单变量问题，也就是在 $a_j$​ 左边查找是否有 $a_i$​=$t−a_j$​，这可以用哈希表维护。我把这个技巧叫做 枚举右，维护左。
